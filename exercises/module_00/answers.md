## Section D: Short Answer

1. **Everything in RAM is erased.** RAM needs power to hold data — when power stops, it forgets instantly.

2. **Storage.** When you save a file, it's written to the MicroSD card (storage) so it stays there even with the power off.

3. Any two of: **Android phones, Mars rovers, the International Space Station, web servers (Google/YouTube/Amazon), Raspberry Pi, supercomputers.**

4. **Hotbar = RAM** (fast, only 9 slots, gone when you quit). **Chest = Storage** (slower, holds lots, stays saved).

5. **1024 ÷ 5 = 204 songs** (approximately). Actually about 204 songs.

## Section E: Trace the Program

```
Step 1: You type "python3 hello.py" in the terminal and press Enter
Step 2: Linux (the OS) receives your command
Step 3: Linux loads the python3 program from storage into RAM
Step 4: Linux loads your hello.py file from storage into RAM
Step 5: The CPU starts reading Python instructions one by one
Step 6: CPU executes print("Hello!") → sends text to terminal
Step 7: Program ends → RAM used by the program is freed
Step 8: Terminal shows the prompt again, ready for your next command
```

- RAM needed: 2 GB (game) + 1 GB (browser) + 0.5 GB (music) = **3.5 GB**
- Available: 4 GB
- **Yes, you can run all three!** 4 - 3.5 = 0.5 GB left over.

- Adding code editor: 3.5 + 0.8 = **4.3 GB needed** — that's MORE than 4 GB.
- **No**, you can't run all four at the same time without the computer slowing down badly (it would use the slower storage as "fake RAM" — called swapping).
